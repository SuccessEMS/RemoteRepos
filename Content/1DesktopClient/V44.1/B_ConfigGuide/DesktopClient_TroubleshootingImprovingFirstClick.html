Troubleshooting: Improving First Click Performance of

If you experience a delay on the "first click" of major functions within the that's too long, there are a couple of solutions. One approach is to leave running all day so that you only experience the first click delay once. Another approach, which is a bit more involved, is to run a function within .NET called NGen. This function pre-compiles .NET applications to make them load significantly faster and eliminate first-click delays.

To Run NGen

 
Note: You must have installed EMS and you must have administrative privileges to perform the following steps.
Warning: While NGen eliminates the first click delay, EMS will not be adding it to our standard installation process; this is an optional workaround you take at your own risk.
  1. Open a command prompt as an administrator. To do this, press the Windows key, type "cmd" without the quotation marks, position the cursor over "Command Prompt" as shown in the screen shot below, right click, and select  "Run as administrator."

  1. At the C:> prompt, go to the directory where EMS is installed. If you have conventionally installed EMS, then it's typically at the following location:

cd \program files (x86)\EMS

However, if you're using the web deployment option to install , then you should find at the following location:

cd %appdata%\EMS2016\EMS

  1. Press Enter and you should see one of the following (where [user name] is your Windows user name):

C:\Program Files (x86)\EMS>

C:\Users\[user name]\AppData\Roaming\EMS2016\EMS>

  1. Type the following:

\windows\microsoft.net\framework64\v4.0.30319\ngen install ems.exe

  1. Press Enter and wait until the process finishes. It will take about 5 minutes. Ignore any error messages.
  2. When the installation finishes, you can close the command prompt and run EMS.
  3. If you install a new or updated version of the Desktop Client, then you have to run NGen again if you want to eliminate the first click delay.
