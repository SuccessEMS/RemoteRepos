Connecting With EMS Platform Services

To begin using {{product_mobile}}, you must first configure it with a valid EMS Platform Services URL. When prompted for a "EMS Server URL", enter the root of EMS Platform Services. For example: If EMS Platform Services is hosted at https://yourcompany.com/EmsPlatform, then enter that URL at the "EMS Server URL" prompt, followed by clicking on the "Connect" link. If the URL is valid, you will be taken to the {{product_mobile}} Sign In page. 

{{product_mobile}} Sign In Invalidation

A mobile user's action sign in session may be invalidated in any of the following ways:

  • Session Expiration, based on the "Token Duration" settings under the Web App > Admin Functions > Mobile App > Persistent Authentication configuration.
  • Changes to security-related information in the user's EMS database account.
    • This includes things like usernames and email addresses, but not things like phone or fax information.
  • Logging into {{product_mobile}} on another device or {{product_DTC}}.

Note: At this time, EMS Platform Services does not allow concurrent sessions.
