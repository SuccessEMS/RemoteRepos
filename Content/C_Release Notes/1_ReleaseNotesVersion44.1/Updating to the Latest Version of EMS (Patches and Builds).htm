Facts About EMS Updates

  • Applying the most recent updates ensures that you are getting the latest fixes and enhancements.
  • Updates are available at the EMS Support Center on www.emssoftware.com.
  • Updates can include database patches and new builds for any EMS components.
  • When an update includes a new build to the EMS Desktop Client, each user machine will need to be updated.
  • When an update includes a new build to the EMS for Outlook client, each user machine will need to be updated.
  • Applying the update will only update the applications you have installed.
  • Updates are usually released on a monthly basis.
  • For each component, updates are cumulative, which means that you can apply an update even if you have not applied the previous update. However, when you update any component to the latest release, you must make sure that you update all components to the latest available release. This requirement can be tricky; please see the example in the box below.
Important: To update to the latest release, be sure to run every Patch and Build available to you on the Software Downloads page since there may be dependencies between Component Builds and related Services. Patches update your database, and Builds update your EMS components. For example, if you have EMS Desktop Client and EMS Web App, you will need the patch (if included in the release) and the builds for EMS Desktop Client and EMS Web App.

At a component level of EMS (for example, EMS for Outlook), you can skip updates. For example, if you're running Update 6 of EMS for Outlook and you want to apply Update 8, you do NOT have to apply Update 7.

However, you must be careful to apply the latest update of every component. For example, say you're running Update 6 of EMS and want to apply Update 8. Because the latest update of one or more components may be in Update 7 instead of Update 8, you have to check Update 7 to make sure you have the latest update of each component.

Download the Latest Update (Patches and Builds)

  1. Review our Release Notes Summary to see a quick list of contents of each Update.
  2. Go to www.emssoftware.com and click Support.
  3. Log in.
  4. Select Software & Docs Library > New Releases & Patches. This page lists downloads available based on your license.
Software & Documents Library

  1. Download the new patches and builds (available based on your license) by selecting the appropriate links in the Download column.

Apply Database Patches

The following steps need to be performed for each System Administrator machine.

  1. On your database server, launch Microsoft SQL Server Management Studio.
IMPORTANT: Back up your EMS database using Microsoft SQL Server Management Studio.
  1. Locate the database patch file you downloaded from the online Support Center (as described above; the file is typically named EMSPatch.sql).
  2. Within Microsoft SQL Server Management Studio, select File > Open > File and open the EMS patch file.
  3. Select your EMS database (not EMS_Master) and execute the query (Query > Execute). When query execution has completed, a “Query Batch Completed” message will display in the Results.
  4. Verify your new Patch Number by launching the EMS Desktop Client and selecting Help > About EMS > Patch Number.

Upgrade Considerations

Consider the following notes depending on your implementation and licensing.

New EMS Desktop Client And/Or Outlook Builds Need to be Updated on Your Users' Machines

You can verify your current Patch Number by launching the EMS Desktop Client and selecting Help > About EMS > Patch Number. If the update includes a new build for the EMS Desktop Client, you will need to install a new EMS build on each user machine. Follow the installation instructions here.

EMS Mobile Downloads

.apk files apply to Android devices, and .ipa files apply to IOS files.

Custom Scripts for HR Toolkit

EMS V44.1 changes how Everyday Web Users are validated. If you have custom HR Toolkit scripts, these need to be tested with V44.1.
