Virtual EMS (VEMS)

Area Description

Create Reservation

Unauthenticated User

Unauthenticated User and Building Hours

Customers were not able to book a space at the opening time of the building due to the 15-minute setup time; however, once a user logged in they were able to do the opposite. Users should not be able to book a space due to the building hours. 

Fix: Modified functionality so that users cannot book the space due to the building hours. 

TFS item 28248

Building Hours Violation Not Working for Unauthenticated Users 

Unauthenticated users who created a reservation with a start time that violated the building's hours of operation did not receive a violation of building hours error message. 

Fix: Modified functionality so that unauthenticated users choosing a start time that violates the building's hours of operation receive an error message. 

TFS item 29832
My Booking Page

Check In Was Not Disappearing from My Booking Page

Once the booking ended in the Mobile Virtual EMS page, the check in icon should have disappeared. 

Fix: Modified functionality so that the check in icon disappears after the booking ends. 

TFS item 30220

APIs. Additional APIs were requested as a feature enhancement.

 

Added the following APIs to the EMS API library.

  • AddServiceOrder: Allows new service orders to be added to bookings.
  • AddServiceOrderDetail: Allows new service order details (resources) to be added to existing service orders
  • UpdateServiceOrder: Allows for edits to service orders.
  • UpdateServiceOrderDetail: Allows for edits to service order details (resources).
  • DeleteServiceOrderDetail: Allows deletion of service order details (resources).
  • DeleteServiceOrder: Allows deletion of service orders.
  • AddReservation5: Allows reservations and bookings tied to a Web User, including Billing Reference/PO Numbers and setup count. 
  • GetCategories: Retrieves category information.
  • GetCategoryStates: Retrieves categories for the building ID entered.
  • GetResources: Retrieves active resource information for the category entered.
  • GetPricingMethods: Retrieves active pricing methods to use with Service Order.
  • ValidateBilling: Checks that Billing Reference, PO Number, or both are valid.
