Application Architecture

The EMS Polycom Interface is comprised of two components:

  1. EMS Polycom Service: Windows service responsible for pushing video conference bookings from EMS to the Polycom Realpresence Resource Manager API at timed intervals.
  2. Polycom Activity DLL: Custom Polycom DLL provided by EMS that will display all EMS Polycom Interface activity within the EMS application, including errors and successfully scheduled video conferences. The Polycom Activity DLL also contains an area that allows EMS Administrators to configure various EMS Polycom Interface settings.

New video conferences booked in EMS are pushed to the Polycom Realpresence Resource Manager API based on various settings defined within EMS (see EMS Polycom Interface Configuration Settings) Subsequent critical booking changes (i.e. date, time or location changes) and/or cancelations are also relayed to the API. The EMS Polycom Interface is a one-way interface and changes made in Polycom will not be reflected in EMS.

See Also: Advanced EMSPolycom Interface Configuration to learn about how data is transferred from EMS to the Polycom Realpresence Resource Manager API.
