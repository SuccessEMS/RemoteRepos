EMS Room Sign App Release Notes V44.1, Update 12

This section describes updates to your EMS software. 

See Also: Contacting Customer Support and Updating to the Latest Version of EMS

Name: V44.1 Update 12

Release Date: June 30, 2017

Patch: 2017.06.29.0

Build #:  17.6.28.1

Enhancements 

Area

Description

ArmorActive Enclosure

Five-Second Tones Now Sent as Best Practice

The audio tones needed to change the LED indicators on the ArmorActive enclosure now play for five seconds.

TFS item 30311

Tones to ArmorActive Enclosures Are Sent Every 45 Seconds

The audio tones needed to change the LED indicators on the ArmorActive enclosure are now sent every 45 seconds, ensuring that the LED color always aligns with the room state.

TFS item 30314


Edit Booking

Users Can Now Extend Their Meetings With One Click

The user experience for extending bookings has been improved and users can now extend their bookings with only one click. See Configuring Edit Booking for more information.

TFS item 29574


Portrait Mode
 

The Room Sign App Can Now Be Deployed in Portrait Mode

The EMS Room Sign App can now be deployed in portrait mode, which displays an agenda view of the day's events. See Deploying the EMS Room Sign App on Tablets for more information.

TFS item 29608

 

Fixes 

Area

Description

Status for Private Booking

Bookings Were Not Appearing if Booked Status Was Status for Private Booking

When users created a profile, set the status to "status for private booking," and created a booking with this profile, the booking did not appear. The availability bar was red and had no text. 

Fix: Modified functionality so that the value in the "Private Booking Label" parameter appeared in the availability bar.

TFS item 30459

Create a Reservation

Large Values of the "Maximum Number of Minutes for New Booking" Parameter Resulted in a Strange Default End Time

Fix: Modified functionality so that any value of the parameter still yielded the proper default end time, and displayed accurate values in the end time picker.

TFS item 30473

Keyboard Was Reopening When Users Hit OK to Confirm a Reservation

The keyboard would reopen when users hit OK to confirm their reservation. 

Fix: Modified functionality so that the keyboard does not reappear when confirming a reservation.

TFS item 25231

 

In Certain Scenarios, Users Could Not Reserve Even if the Room Was Available

Fix: Modified functionality so that users can reserve a room as long as the room is available and the profile allows reservations. 

TFS item 30593
